THERON_USE_BOOST_THREADS
#define THERON_USE_BOOST_THREADS   0

Controls whether Boost threads, locks, etc are used.

If THERON_USE_BOOST_THREADS is defined as 0 then Windows threads are used, introducing a dependency on the windows.h header. Otherwise Boost threads are used, introducing a dependency on boost::thread.

Defaults to 0 (Windows threads).

The value of THERON_USE_BOOST_THREADS can be overridden by defining it globally in the build (in the makefile using -D, or in the project preprocessor settings in Visual Studio).

Note:
The makefile build defines THERON_USE_BOOST_THREADS as 1 by default, overriding the default value defined in the code. This is arranged this way on the assumption that users who are using the makefile build are likely to want to avoid a dependency on Windows. The value defined by the makefile can be controlled by specifying threads=boost or threads=windows on the make command line.
